联系人 [待校准@6167]

不同类型的问题或讨论对应不同的通讯途径; 请查看以下描述以确保您选择了正确的方法。 [待校准@6168]

支持 [待校准@6169]

需要帮助故障诊断工具系统？首先，搜索 ROS Answers ，看看其他人是否有类似的问题，以及他们的解决方案是否适合你。 [待校准@6170]

如果没有，问一个关于 ROS Answers 的新问题。确保添加标签，至少是你正在运行的 ros2 标签和发行版，例如 foxy 。如果您的问题与此处的文档相关，请添加一个标签，如 docs ，或更具体地调用 tutorials 。 [待校准@6171]

提供支持 [待校准@6172]

ROS 2用户来自广泛的技术背景，使用各种不同的操作系统，并且不一定具有ROS (1或2) 的任何先前经验。因此，对于拥有丰富经验的用户来说，提供支持非常重要。 [待校准@6173]

如果你在 ROS Answers 上看到一个与你自己遇到的问题相似的问题，请考虑提供一些对你的情况有帮助的建议。如果你不确定你的回答是否正确，不要担心。只需这样说，其他社区成员将在必要时加入。 [待校准@6174]

问题 [待校准@6175]

如果您发现错误、有改进建议或特定于一个包的问题，您可以在GitHub上打开一个问题。 [待校准@6176]

例如，如果你正在遵循 tutorials here ，并且遇到一个在你的系统上不起作用的指令，你可以在 ros2_documentation 回购中打开一个问题。 [待校准@6177]

您可以在 ROS 2's GitHub 上搜索单个ROS 2仓库。 [待校准@6178]

在打开问题之前，通过在ros2和amentGitHub组织中搜索，检查其他用户是否报告了类似的问题: example search query 。 [待校准@6179]

如果尚未报告，请随时在相应的仓库跟踪器中打开问题。如果不清楚在特定问题上使用哪个跟踪器，请将其归档到 ros2/ros2 repository 中，我们将对其进行查看。 [待校准@6180]

提交问题时，请确保: [待校准@6181]

  • 包括足够的信息让其他人理解这个问题。 [待校准@6182]

  • 包括与问题相关的确切平台、软件、版本和环境的信息。这包括您如何安装软件 (从二进制文件或源文件) 以及您使用的ROS中间件/DDS供应商 (如果您知道的话)。 [待校准@6183]

  • 如果发生错误，考虑提供 short, self contained, correct (compilable), example 。 [待校准@6184]

拉取请求 [待校准@6185]

当您感到足够舒适，可以直接建议对代码进行特定更改时，您可以提交拉取请求。任何 the ros2 repositories 都欢迎拉取请求。有关如何做出贡献的更多详细信息和礼节，请参阅 Contributing 页面。 [待校准@6186]

讨论 [待校准@6187]

要开始与其他ROS 2社区成员的讨论，请访问官方 ROS Discourse 。演讲的内容应该是高水平的; 这不是一个回答代码问题的地方，但是开始讨论最佳实践或提高标准是合适的。 [待校准@6188]

关于ROS 2开发版本和计划的讨论正在 “Next Generation ROS” Discourse category 进行。参与这些讨论是就ROS 2的不同功能将如何工作和实施发表意见的重要方式。 [待校准@6189]

ROS生态系统背后的多样化社区是其最大的资产之一。我们鼓励ROS社区的所有成员参与这些设计讨论，以便我们能够利用社区成员的经验，并牢记ROS的各种用例。 [待校准@6190]

私人联系人 [待校准@6191]

如果您想私下与我们联系 (例如g.，如果您的问题包含对您的组织或项目敏感的信息，或者与安全问题有关)，您可以直接通过 “ros@osrfoundation.org” 给我们发送电子邮件。 [待校准@6192]
