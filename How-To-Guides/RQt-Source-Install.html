从源文件构建RQt [待校准@883]

我们提供我们开发版本设置来帮助未来用户简易伸缩RQt创建自己的插件。我们鼓励你贡献这些插件回 ros-visualization GitHub仓库! [待校准@884]

系统要求 [待校准@885]

这些说明是为Crystal Clemmys的目标平台编写的 (见 REP )。 [待校准@886]

  • Ubuntu仿生海狸18.04 64位 [待校准@887]

  • Mac OSX斯莱瑞10.12.x [待校准@888]

  • 带有Visual Studio 2017的Windows 10 [待校准@889]

其他要求 [待校准@890]

  • 在ROS 2Crystal中，最小Qt版本是 Qt5 [待校准@891]

从源头构建 [待校准@892]

为了从源文件构建RQt，首先在 “~/ros2_ws/” 处创建一个ROS 2工作区。这一步已经被 building ROS 2 from source instructions 覆盖了，所以我们在这里跳过它。 [待校准@893]

下载RQt存储库 [待校准@894]

cd ~/ros2_ws
wget https://raw.githubusercontent.com/PickNikRobotics/rqt2_setup/master/rqt2.repos
vcs import src --force < rqt2.repos

作为托管 .repos 文件的替代，您可以使用 rosinstall_generator 生成自定义文件: [待校准@895]

rosinstall_generator --rosdistro crystal --upstream-development --repos python_qt_binding qt_gui_core rqt <more-repos-with-rqt-plugins> > rqt2.repos

安装依赖项 [待校准@896]

对于非Linux平台，请在此处继续之前参见 macOS RQt source install page or the Windows 10 RQt source install page 。 [待校准@897]

rosdep install --from-paths src --ignore-src --rosdistro bouncy -y --skip-keys "console_bridge fastcdr fastrtps rti-connext-dds-5.3.1 urdfdom_headers"

构建工作空间 [待校准@898]

通常，构建工作区非常简单: [待校准@899]

colcon build

对于Windows，建议使用 --merge-install 选项。 [待校准@900]

colcon build --merge-install

高级Colcon用途: [待校准@901]

  • 在控制台上显示详细输出: [待校准@902]

    colcon build –event-handlers console_direct+

  • 仅构建一个包及其依赖项: [待校准@903]

    colcon build –packages-up-to rqt_shell

采购您的环境 [待校准@904]

Linux或macOS [待校准@905]

. install/local_setup.bash

Windows [待校准@588]

call install/local_setup.bat

见 Overview of RQt 。 [待校准@907]
