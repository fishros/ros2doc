系统要求 [待校准@5888]

我们支持macOS 10.14 (Mojave)。 [待校准@6118]

但是，一些新版本 (如10.13.x) 和一些旧版本 (如10.11.x和10.10.x) 也可以使用。 [待校准@6119]

安装先决条件 [待校准@6120]

您需要安装以下内容来构建ROS 2: [待校准@6121]

  1. Unix [待校准@6122]

    • 如果尚未安装，请安装Xcode。 [待校准@6123]

    • 注意: 11.3.1之后的版本无法再安装在macOS Mojave上，因此您需要手动安装旧版本，请参阅: https://stackoverflow.com/a/ 61046761 [待校准@6124]

    • 另外，如果尚未安装它，请安装命令行工具: [待校准@6125]

      xcode-select --install
# This command will not succeed if you have not installed Xcode.app
sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
# If you installed Xcode.app manually, you need to either open it or run:
sudo xcodebuild -license
# To accept the Xcode.app license

  2. 酿造 * (需要安装更多的东西; 你可能已经有了这个) *: [待校准@6126]

    • 按照安装说明在http://brew.sh/ [待校准@6127]

    • Optional: Check that brew is happy with your system configuration by running:

      brew doctor

      修复它识别的任何问题。 [待校准@6129]

  3. 使用 brew 安装更多材料: [待校准@6130]

    brew install cmake cppcheck eigen pcre poco tinyxml wget bullet

brew install python@3.8
brew unlink python
# Make the python command be Python 3.8
brew link --force python@3.8

# install dependencies for Fast-RTPS if you are using it
brew install asio tinyxml2

brew install opencv

# install console_bridge for rosbag2
brew install console_bridge

# install OpenSSL for DDS-Security
brew install openssl
# if you are using ZSH, then replace '.bashrc' with '.zshrc'
echo "export OPENSSL_ROOT_DIR=$(brew --prefix openssl)" >> ~/.bashrc

# install dependencies for rcl_logging
brew install log4cxx spdlog

# install CUnit for Cyclone DDS
brew install cunit

  4. 安装rviz依赖项 [待校准@6131]

    # install dependencies for Rviz
brew install qt@5 freetype assimp

# Add the Qt directory to the PATH and CMAKE_PREFIX_PATH
export CMAKE_PREFIX_PATH=$CMAKE_PREFIX_PATH:/usr/local/opt/qt@5
export PATH=$PATH:/usr/local/opt/qt@5/bin

  5. 安装rqt依赖项 [待校准@6132]

    brew install graphviz pyqt5 sip

  6. 使用 python3 -m pip ( pip 可能安装Python3或Python2) 安装更多: [待校准@6133]

    python3 -m pip install -U \
 argcomplete catkin_pkg colcon-common-extensions coverage \
 cryptography empy flake8 flake8-blind-except flake8-builtins \
 flake8-class-newline flake8-comprehensions flake8-deprecated \
 flake8-docstrings flake8-import-order flake8-quotes ifcfg \
 importlib-metadata lark-parser lxml mock mypy==0.761 netifaces \
 nose pep8 pydocstyle pydot pygraphviz "pyparsing>=2.4,<3" \
 pytest-mock rosdep rosdistro setuptools vcstool

    请确保 “$ PATH'' 环境变量包含二进制文件的安装位置 (默认值:“ $ HOME/Library/Python/<version>/bin'') [待校准@6134]

  7. Optional: if you want to build the ROS 1<->2 bridge, then you must also install ROS 1:

    • 从正常安装开始说明: https://wiki.ros.org/kinetic/Installation/OSX/Homebrew/Source [待校准@6136]

    • 当您到达调用 rosinstall_generator 来获取源文件的步骤时，这里有一个替代调用，它只带来生成有用的bridge所需的最低限度: [待校准@6137]

      rosinstall_generator catkin common_msgs roscpp rosmsg --rosdistro kinetic --deps --wet-only --tar > kinetic-ros2-bridge-deps.rosinstall
wstool init -j8 src kinetic-ros2-bridge-deps.rosinstall

      否则，只需遵循正常说明，然后在继续构建ROS 2之前源文件生成的 install_isolated/setup.bash 。 [待校准@6138]

禁用系统完整性保护 (SIP) [待校准@6139]

macOS/OS X版本> = 10.11默认情况下启用了系统完整性保护。使SIP不防止进程继承动态链接器环境变量，如 DYLD_LIBRARY_PATH ，你需要禁用它 following these instructions _。 [待校准@6140]

获取ROS 2代码 [待校准@6011]

创建工作区并克隆所有存储库: [待校准@5904]

mkdir -p ~/ros2_foxy/src
cd ~/ros2_foxy
wget https://raw.githubusercontent.com/ros2/ros2/foxy/ros2.repos
vcs import src < ros2.repos

安装其他DDS供应商 (可选) [待校准@6141]

如果你想使用另一个DDS或RTPS供应商，除了默认的，eProsima的快速RTPS，你可以找到说明 here 。 [待校准@5908]

构建ROS 2代码 [待校准@6016]

运行 colcon 工具来构建一切 (更多关于在 this tutorial 中使用 colcon 的信息): [待校准@6142]

cd ~/ros2_foxy/
colcon build --symlink-install --packages-skip-by-dep python_qt_binding

注意: 由于SIP、Qt @ 5和PyQt5的问题尚未解决，我们需要禁用 python_qt_binding 才能成功构建。这将删除问题解决后，看到: https://github.com/ros可视化/python_qt_binding/问题/103 [待校准@6143]

环境设置 [待校准@5913]

获取ROS 2安装文件: [待校准@6144]

. ~/ros2_foxy/install/setup.bash

这将自动调用y为任何为其构建支持的DDS供应商设置环境。 [待校准@6023]

尝试一些例子 [待校准@5916]

在一个终端中，如上所述设置ROS 2环境，然后运行c talker : [待校准@6145]

ros2 run demo_nodes_cpp talker

在另一个终端源文件中，安装文件，然后运行Python listener : [待校准@5957]

ros2 run demo_nodes_py listener

你应该看到 talker 说这是 Publishing 消息 listener 说 I heard 这些消息。这将验证capis和Python api是否正常工作。万岁! [待校准@5919]
