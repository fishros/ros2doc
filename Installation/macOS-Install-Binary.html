本页介绍如何从预构建的二进制包在macOS上安装ROS 2。 [待校准@6149]

备注

预构建的二进制文件不包括所有ROS 2包。包括 ROS base variant 中的所有包，并且只包括 ROS desktop variant 中的包的一个子集。包的准确列表由列在 this ros2.repos file 列的仓库描述。 [待校准@6150]

系统要求 [待校准@5888]

我们支持macOS Mojave (10.14)。 [待校准@6151]

安装先决条件 [待校准@6053]

在安装ROS 2之前，您需要安装以下内容。 [待校准@6152]

  • 酿造 * (需要安装更多的东西; 你可能已经有了这个) *: [待校准@6126]

    • 按照安装说明在http://brew.sh/ [待校准@6127]

    • Optional: Check that brew is happy with your system configuration by running:

        brew doctor


Fix any problems that it identifies.

  • 使用 brew 安装更多材料: [待校准@6130]

    brew install python@3.8
# Unlink in case you have python@3.7 installed already
brew unlink python
# Make the python command be Python 3.8
brew link --force python@3.8

# install asio and tinyxml2 for Fast-RTPS
brew install asio tinyxml2

# install dependencies for robot state publisher
brew install tinyxml eigen pcre poco

# OpenCV isn't a dependency of ROS 2, but it is used by some demos.
brew install opencv

# install OpenSSL for DDS-Security
brew install openssl
# if you are using ZSH, then replace '.bashrc' with '.zshrc'
echo "export OPENSSL_ROOT_DIR=$(brew --prefix openssl)" >> ~/.bashrc

# install Qt for RViz
brew install qt freetype assimp

# install console_bridge for rosbag2
brew install console_bridge

# install dependencies for rcl_logging_log4cxx
brew install log4cxx spdlog

# install CUnit for Cyclone DDS
brew install cunit

  • 安装ros2医生依赖项 [待校准@6153]

    python3 -m pip install rosdistro [待校准@6154]

  • 安装rqt依赖项 [待校准@6132]

    brew install sip pyqt5 [待校准@6155]

    在安装过程中查找sip内容时，修复一些路径名 (参见 ROS 1 wiki ): [待校准@6156]

    ln -s /usr/local/share/sip/Qt5 /usr/local/share/sip/PyQt5 [待校准@6157]

    brew install graphviz [待校准@6158]

    python3 -m pip install pygraphviz pydot [待校准@6159]

    备注

    你可能会遇到安装 pygraphviz 、 "error: Error locating graphviz" 的问题。请尝试以下install命令: [待校准@6160]

    python3 -m pip install --install-option="--include-path=/usr/local/include/" --install-option="--library-path=/usr/local/lib/" pygraphviz

  • 安装SROS2依赖项 [待校准@6161]

    python3 -m pip install lxml [待校准@6162]

  • 为命令行工具安装其他运行时依赖项: [待校准@6163]

    python3 -m pip install catkin_pkg empy ifcfg lark-parser lxml netifaces numpy pyparsing pyyaml setuptools argcomplete

禁用系统完整性保护 (SIP) [待校准@6139]

macOS/OS X版本> = 10.11默认情况下启用了系统完整性保护。使SIP不防止进程继承动态链接器环境变量，如 DYLD_LIBRARY_PATH ，你需要禁用它 following these instructions _。 [待校准@6140]

下载ROS 2 [待校准@5943]

  • 去发行版本页面: https://github.com/ros2/发行版本 [待校准@6099]

  • 下载macOS的最新软件包; 让我们假设它最终为 “~/下载/ros2-release-distro-date-macos-amd64.tar.bz2”。 [待校准@6164]

    • 注意: 可能有多个二进制下载选项，这可能会导致文件名不同。 [待校准@5946]

  • 打开包装: [待校准@5947]

    mkdir -p ~/ros2_foxy
cd ~/ros2_foxy
tar xf ~/Downloads/ros2-release-distro-date-macos-amd64.tar.bz2

尝试一些例子 [待校准@5916]

在一个终端中，如上所述设置ROS 2环境，然后运行c talker : [待校准@6145]

ros2 run demo_nodes_cpp talker

在另一个终端中，设置ROS 2环境，然后运行Python listener : [待校准@6165]

ros2 run demo_nodes_py listener

你应该看到 talker 说这是 Publishing 消息 listener 说 I heard 这些消息。这将验证capis和Python api是否正常工作。万岁! [待校准@5919]

卸载 [待校准@5934]

  1. 如果你按照上面的指示用colcon安装了你的工作区， "uninstalling" 可能只是打开一个新的终端，而不是采购工作区的 setup 文件。这样，您的环境就会表现得好像您的系统上没有安装Foxy。 [待校准@5935]

  2. 如果您还尝试释放空间，则可以使用以下命令删除整个工作区目录: [待校准@5936]

    rm -rf ~/ros2_foxy
