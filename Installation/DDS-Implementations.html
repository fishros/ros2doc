安装DDS实现 [待校准@5305]

默认情况下，ROS 2使用DDS作为其 middleware 。它与多个DDS或RTPS (DDS有线协议) 供应商兼容。目前支持eProsima的Fast DDS、RTI的Connext DDS、EclipseCycloneDDS和GurumNetworks GurumDDS。有关按发行版支持的DDS供应商，请参阅https://ros.org/reps/ rep-2000.html。 [待校准@5306]

在Foxy中，默认的DDS供应商是eProsima的Fast DDS。 [待校准@5307]

  • [需手动修复的语法] Working with Eclipse Cyclone DDS 解释了如何利用CycloneDDS。 [待校准@5308]

  • [需手动修复的语法] Working with eProsima Fast DDS 解释了如何利用Fast DDS。 [待校准@5309]

  • [需手动修复的语法] Working with GurumNetworks GurumDDS 解释了如何利用古鲁姆斯。 [待校准@5310]

如果您想使用其他供应商之一，则需要在构建之前单独安装他们的软件。ROS 2构建将自动调用对已正确安装和源文件的供应商的构建支持。 [待校准@5311]

安装新的DDS供应商后，您可以更改运行时使用的供应商: Working with Multiple RMW Implementations 。 [待校准@5312]

下面提供了安装其他DDS供应商的详细说明。 [待校准@5313]

Ubuntu Linux源文件安装 [待校准@5315]

RTI Connext (版本5.3.1，仅限amd64) [待校准@5316]

ROS 2 apt仓库提供的Debian包 [待校准@5317]

您可以在ROS 2 apt仓库安装RTI Connext的Debian包。您需要接受RTI的许可。 [待校准@5318]

sudo apt install -q -y \
    rti-connext-dds-5.3.1  # from packages.ros.org/ros2/ubuntu

获取安装文件以设置 NDDSHOME 环境变量。 [待校准@5319]

cd /opt/rti.com/rti_connext_dds-5.3.1/resource/scripts && source ./rtisetenv_x64Linux3gcc5.4.0.bash; cd -

注意: 使用 zsh 时，您需要在采购时位于脚本的目录中，以使其正常工作 [待校准@5320]

现在，您可以正常构建，并且也将构建对RTI的支持。 [待校准@5321]

来自RTI的官方二进制包 [待校准@5322]

你可以安装Connext 5.3.1包装Linux提供RTI，通过选项 university, purchase or evaluation [待校准@5323]

下载后，在 .run 可执行文件上使用 “chmod + x”，然后执行它。请注意，如果要安装到系统目录，也请使用 sudo 。 [待校准@5324]

默认位置为 “~/rti_connext_dds-5.3.1” [待校准@5325]

安装后，运行RTIlauncher并将其指向您的许可证文件 (从RTI获得)。 [待校准@5326]

将以下行添加到您的 .bashrc 文件中，指向您的许可证副本。 [待校准@5327]

export RTI_LICENSE_FILE=path/to/rti_license.dat

获取安装文件以设置 NDDSHOME 环境变量。 [待校准@5319]

source ~/rti_connext_dds-5.3.1/resource/scripts/rtisetenv_x64Linux3gcc5.4.0.bash

现在，您可以正常构建，并且也将构建对RTI的支持。 [待校准@5321]

Ubuntu Linux二进制安装 [待校准@5328]

RTI Connext (版本5.3.1，仅限amd64) [待校准@5316]

要使用RTI Connext DDS，有适用于 university, purchase or evaluation or you can install a libraries-only Debian package of RTI Connext 5.3.1, available from the OSRF Apt repository under a non-commercial license __ 的全套安装选项。 [待校准@5329]

安装libs-only Debian包: [待校准@5330]

sudo apt update && sudo apt install -q -y rti-connext-dds-5.3.1

您需要接受RTI的许可协议，并在安装中找到 “rti_license.Dat” 文件。 [待校准@5331]

将以下行添加到您的 .bashrc 文件中，指向您的许可证副本 (及其源文件)。 [待校准@5332]

export RTI_LICENSE_FILE=path/to/rti_license.dat

所有选项都需要您源文件以设置 NDDSHOME 环境变量: [待校准@5333]

cd /opt/rti.com/rti_connext_dds-5.3.1/resource/scripts && source ./rtisetenv_x64Linux3gcc5.4.0.bash; cd -

注意: 以上内容可能需要修改以匹配您的RTI安装位置 [待校准@5334]

如果你想安装Connext DDS-安全插件，请参考 this page 。 [待校准@5335]
