本指南介绍了如何在Windows上为ROS 2设置开发版本环境。 [待校准@5974]

先决条件 [待校准@5798]

首先按照二进制安装页面上的 Installing Prerequisites 步骤操作。 [待校准@5975]

当你到达 "Downloading ROS 2" 区时，停下来返回这里。 [待校准@5976]

语言支持 [待校准@5977]

确保你有一个支持 UTF-8 的区域。例如，对于中文Windows 10安装，您可能需要安装 English language pack 。 [待校准@5978]

附加先决条件 [待校准@5979]

从源文件构建时，您需要安装一些额外的先决条件。 [待校准@5980]

从Chocolatey安装额外的先决条件 [待校准@5981]

首先安装git: [待校准@5982]

choco install -y git

您需要将Git cmd文件夹 “c: 程序文件 Git cmd'” 附加到路径中 (您可以通过单击Windows图标，键入 "Environment Variables" ，然后点击 "Edit the system environment variables" 。在生成的对话框中，单击 "Environment Variables" ，在底部窗格中单击 "Path" ，然后单击 "Edit" 并添加路径)。 [待校准@5983]

安装开发人员工具 [待校准@5984]

现在，我们准备安装一些用于帮助开发ROS 2的工具。 [待校准@5985]

让我们从 vcstool 开始: [待校准@5986]

pip install -U vcstool

你可以通过运行 vcs 来测试它 (你应该可以在同一个cmd提示中这样做)。 [待校准@5987]

接下来，安装 colcon : [待校准@5988]

pip install -U colcon-common-extensions

你可以通过运行 colcon 来测试它 (你应该可以在同一个cmd提示中这样做)。 [待校准@5989]

另外，你应该安装 curl : [待校准@5990]

choco install -y curl

安装依赖项 [待校准@5991]

接下来安装最新版本的 setuptools 和 pip : [待校准@5992]

python -m pip install -U setuptools pip

然后，您可以继续安装其他Python依赖项: [待校准@5993]

pip install -U catkin_pkg cryptography EmPy ifcfg lark-parser lxml numpy pyparsing pyyaml

接下来安装测试工具，如 pytest 等: [待校准@5994]

pip install -U pytest pytest-mock coverage mock

接下来安装像 flake8 和其他的油缸和格子: [待校准@5995]

pip install -U flake8 flake8-blind-except flake8-builtins flake8-class-newline flake8-comprehensions flake8-deprecated flake8-docstrings flake8-import-order flake8-quotes mypy==0.761 pep8 pydocstyle

接下来安装cppcheck: [待校准@5996]

choco install -y cppcheck

接下来安装xmllint: [待校准@5997]

  • 从https://www.zlatkovic.com/projects/libxml/ 下载 libxml2 的 64 bit binary archives (及其属地 iconv 和 zlib ) [待校准@5998]

  • 打开所有archives到例如 ''c: xmllint'' [待校准@5999]

  • 将 “c: xmllint bin” 添加到 PATH 。 [待校准@6000]

安装Qt5 [待校准@6001]

仅当您正在构建rviz时才需要此部分，但它附带了我们的默认源文件集，因此如果您不知道，则假设您正在构建它。 [待校准@6002]

首先从Qt的网站获取安装程序: [待校准@6003]

https://www.qt.io/download [待校准@6004]

选择开源版本，然后选择 Qt Online Installer for Windows 。 [待校准@6005]

运行安装程序并安装qt5。 [待校准@6006]

我们建议您将其安装到默认位置 “c: qt”，但是如果您选择其他位置，请确保相应地更新以下路径。当选择要安装的组件时，Foxy和更高版本绝对需要的唯一东西是 Qt -> Qt 5.15.0 树下合适的MSVC 64位组件。在撰写本文档时，我们正在使用 5.15.0 ，这's what we recommend since that'我们在Windows上测试的全部内容，但以后的版本可能也会起作用。对于Foxy及以后，一定要选择 MSVC 2019 64-bit 。之后，默认设置就可以了。 [待校准@6007]

最后，设置 Qt5_DIR 环境变量 cmd.exe 你打算建立使CMake可以找到: [待校准@6008]

set Qt5_DIR=C:\Qt\5.15.0\msvc2019_64

您可以使用 “setx-m Qt5_DIR C :Qt 5.15.0 msvc2019_64” 永久设置它，但这需要管理员。 [待校准@6009]

备注

此路径可能会根据您使用的MSVC版本或是否将其安装到其他目录而更改。 [待校准@6010]

获取ROS 2代码 [待校准@6011]

现在我们有了开发版本工具，我们可以获取ROS 2源文件代码。 [待校准@6012]

首先设置一个开发版本文件夹，例如 “c: dev ros2_foxy”: [待校准@6013]

md \dev\ros2_foxy\src
cd \dev\ros2_foxy

获取定义要克隆的仓库的 ros2.repos 文件: [待校准@6014]

# CMD
curl -sk https://raw.githubusercontent.com/ros2/ros2/foxy/ros2.repos -o ros2.repos

# PowerShell
curl https://raw.githubusercontent.com/ros2/ros2/foxy/ros2.repos -o ros2.repos

接下来，您可以使用 vcs 导入 ros2.repos 文件中列出的仓库: [待校准@6015]

# CMD
vcs import src < ros2.repos

# PowerShell
vcs import --input ros2.repos src

安装额外的DDS实现 (可选) [待校准@5907]

如果你想使用另一个DDS或RTPS供应商，除了默认的，eProsima的快速RTPS，你可以找到说明 here 。 [待校准@5908]

构建ROS 2代码 [待校准@6016]

要构建ROS 2，您需要以管理员身份运行Visual Studio命令提示符 ( "x64 Native Tools Command Prompt for VS 2019" )。 [待校准@6017]

快速RTPS与ROS 2源文件捆绑在一起，除非您将 AMENT_IGNORE 文件放在 “src eProsima” 文件夹中，否则它将始终被构建。 [待校准@6018]

要构建 “dev ros2_foxy” 文件夹树: [待校准@6019]

colcon build --merge-install

备注

我们在这里使用 --merge-install 来避免在构建结束时太长的 PATH 变量。如果要调整这些说明以构建较小的工作区，则可以使用默认行为，即隔离安装，即每个包安装到不同的文件夹的位置。 [待校准@6020]

备注

如果您正在进行调试构建，请使用 python_d path\to\colcon_executable build --cmake-args -DCMAKE_BUILD_TYPE=Debug. See Extra stuff for debug mode 了解有关在Windows上的调试构建中运行Python代码的更多信息。 [待校准@6021]

环境设置 [待校准@5913]

启动命令shell并源文件ROS 2安装文件以设置工作区: [待校准@6022]

call C:\dev\ros2_foxy\install\local_setup.bat

这将自动调用y为任何为其构建支持的DDS供应商设置环境。 [待校准@6023]

正常情况下，如果没有其他错误，则前一个命令输出 “系统找不到指定的路径。” 恰好一次。 [待校准@6024]

测试和运行 [待校准@6025]

请注意，首次运行任何可执行文件时，必须允许通过Windows防火墙弹出窗口访问网络。 [待校准@6026]

您可以使用以下命令运行测试: [待校准@6027]

colcon test --merge-install

备注

只有当它也用于构建步骤时，才使用``--merge-install`` should。 [待校准@6028]

之后，您可以使用以下命令获取测试摘要: [待校准@6029]

colcon test-result

要运行这些示例，首先打开一个干净的新 cmd.exe ，并通过采购 local_setup.bat 文件来设置工作区。然后，运行c talker : [待校准@6030]

call install\local_setup.bat
ros2 run demo_nodes_cpp talker

在单独的shell中，您可以执行相同的操作，而是运行Python listener : [待校准@6031]

call install\local_setup.bat
ros2 run demo_nodes_py listener

你应该看到 talker 说这是 Publishing 消息 listener 说 I heard 这些消息。这将验证capis和Python api是否正常工作。万岁! [待校准@5919]

备注

不建议在相同的cmd提示中构建已为 local_setup.bat 源文件。 [待校准@6032]

调试模式的额外内容 [待校准@6033]

如果您希望能够在调试模式下运行所有测试，您需要安装更多内容: [待校准@6034]

  • 为了能够提取Python源文件tarball，您可以使用PeaZip: [待校准@6035]

choco install -y peazip

  • 您还需要SVN，因为某些Python源文件构建依赖项是通过SVN签出的: [待校准@6036]

choco install -y svn hg

  • 安装上述内容后，您需要退出并重新启动命令提示符。 [待校准@6037]

  • 从 tgz 中获取并提取Python 3.8.3源文件: [待校准@6038]

    • https://www.python.org/ftp/python/ 3.8.3/Python-3.8.3.tgz [待校准@6039]

    • 为了使这些说明简洁，请解压缩到 “c: dev Python-3.8.3” [待校准@6040]

  • 现在，从Visual Studio命令提示符以调试模式构建Python源文件: [待校准@6041]

cd C:\dev\Python-3.8.3\PCbuild
get_externals.bat
build.bat -p x64 -d

  • 最后，将构建产品复制到发布模式Python可执行文件和DLL旁边的Python 38安装目录中: [待校准@6042]

cd C:\dev\Python-3.8.3\PCbuild\amd64
copy python_d.exe C:\Python38 /Y
copy python38_d.dll C:\Python38 /Y
copy python3_d.dll C:\Python38 /Y
copy python38_d.lib C:\Python38\libs /Y
copy python3_d.lib C:\Python38\libs /Y
copy sqlite3_d.dll C:\Python38\DLLs /Y
for %I in (*_d.pyd) do copy %I C:\Python38\DLLs /Y

  • 现在，从新的命令提示符，确保 python_d 工作: [待校准@6043]

python_d -c "import _ctypes ; import coverage"

  • 验证 python_d 的操作后，有必要使用启用调试的库重新安装一些依赖项: [待校准@6044]

python_d -m pip install --force-reinstall https://github.com/ros2/ros2/releases/download/numpy-archives/numpy-1.18.4-cp38-cp38d-win_amd64.whl
python_d -m pip install --force-reinstall https://github.com/ros2/ros2/releases/download/lxml-archives/lxml-4.5.1-cp38-cp38d-win_amd64.whl

  • 验证这些依赖项的安装: [待校准@6045]

python_d -c "from lxml import etree ; import numpy"

  • 当您希望返回到构建发行二进制文件时，必须卸载调试变体并使用发行变体: [待校准@6046]

python -m pip uninstall numpy lxml
python -m pip install numpy lxml

  • 要创建可执行文件python脚本 (.Exe)，应使用python _ d调用colcon以及相应的CMake构建类型。如果使用pip安装colcon，则可以通过 pip show colcon-core 找到colcon可执行文件的路径。 [待校准@6047]

python_d path\to\colcon_executable build --merge-install --cmake-args -DCMAKE_BUILD_TYPE=Debug

  • 万岁，你完蛋了! [待校准@6048]

卸载 [待校准@5934]

    rmdir /s /q \ros2_foxy
