Bouncy Bolson is the second release of ROS 2.

支持的平台 [待校准@1387]

此版本的ROS 2在四个平台上受支持 (有关详细信息，请参见 REP 2000 ): [待校准@4132]

  • Ubuntu 18.04 (仿生) [待校准@4133]

    • 适用于amd64和arm64的Debian包 [待校准@4134]

  • Ubuntu 16.04 (Xenial) [待校准@4092]

    • 不支持Debian包，但支持从源文件构建 [待校准@4135]

  • Mac macOS 10.12斯莱瑞) [待校准@4093]

  • 带有Visual Studio 2017的Windows 10 [待校准@4136]

提供了二进制包以及如何从源文件编译的说明 (见 install instructions as well as documentation )。 [待校准@4137]

特征 [待校准@1389]

此ROS 2版本中的新功能 [待校准@4096]

  • [需手动修复的语法] New launch system 具有更强大和更灵活的Python API。 [待校准@4138]

  • 参数可以作为 command line arguments 传递给C++ 可执行文件。 [待校准@4139]

  • 通过 command line arguments. 进行静态重映射 [待校准@4140]

  • 对Python客户端库的各种改进。 [待校准@4141]

  • 支持发布和订阅序列化数据。这是即将开展的本地rosbag实施工作的基础。 [待校准@4142]

  • 更多的 command line tools ，例如用于处理参数和生命周期状态。 [待校准@4143]

  • 默认情况下，二进制包/fatarchive支持三个RMW实现 (无需从源文件构建): [待校准@4144]

    • [需手动修复的语法]eProsima的快速RTPS (默认) [待校准@4145]

    • 项目完结，系统自动填充内容 [待校准@4146]

    • ADLINK的OpenSplice [待校准@4147]

有关所有可用功能的概述，包括早期发行版本的功能，请参阅 Features 页面。 [待校准@4148]

自Ardent发布以来的变化 [待校准@4149]

自 Ardent Apalone 发布以来的变化: [待校准@4150]

  • 蟒蛇包 launch 已经重新设计。以前的Python API已移动到子模块 launch.legacy 中。如果不需要过渡到新的Python API，您可以更新现有的launch文件以继续使用旧API。 [待校准@4151]

  • 包含命名空间的ROS话题名称映射到DDS话题，包括它们的命名空间。DDS分区不再用于此。 [待校准@4152]

  • 推荐的构建工具现在是 colcon 而不是 ament_tools 。该开关没有针对每个ROS 2包中的代码的 implications 安装说明已更新， read-the-docs page 描述了如何将现有的 ament_tools 调用映射到 colcon 。 [待校准@4153]

  • [需手动修复的语法] this rclcpp::Node::create_subscription() signature 的参数顺序已被修改。 [待校准@4154]

已知问题 [待校准@4127]

  • 新型launch文件 may hang on shutdown 一些组合平台RMW实施。 [待校准@4155]

  • 当寻址到特定节点时，名称空间的静态重新映射为 not working correctly 。 [待校准@4156]

  • [需手动修复的语法] Opensplice error messages may be printed 使用 ros2 param 和 ros2 lifecycle 命令行工具。 [待校准@4157]
