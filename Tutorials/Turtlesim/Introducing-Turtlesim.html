背景 [待校准@6276]

Turtlesim是学习ROS 2的轻量级模拟器。它说明了ROS 2在最基本的层面上做了什么，让你知道你以后将如何处理一个真正的机器人或机器人仿真。 [待校准@8430]

rqt是ROS 2的GUI工具。在rqt中完成的所有操作都可以在命令行上完成，但是它提供了一种更简单、更用户友好的方式来操作ROS 2元素。 [待校准@8431]

本教程涉及核心ROS 2概念，如节点、话题和服务的分离。所有这些概念将在后面的教程中详细阐述; 现在，你将简单地设置工具并感受它们。 [待校准@8432]

先决条件 [待校准@6278]

上一教程 配置ROS 2环境 [待校准@6646] 将向您展示如何设置您的环境。 [待校准@8433]

任务 [待校准@6285]

1安装turtlesim [待校准@8434]

如往常一样，从在新终端采购安装文件开始，如 previous tutorial 所述。 [待校准@8435]

为您的ROS 2发行版安装turtlesim包: [待校准@8436]

sudo apt update

sudo apt install ros-foxy-turtlesim

只要您安装的ROS 2中的archive包含 ros_tutorials 仓库，您就应该已经安装了turtlesim。 [待校准@8437]

只要您安装的ROS 2中的archive包含 ros_tutorials 仓库，您就应该已经安装了turtlesim。 [待校准@8437]

检查软件包是否已安装: [待校准@8438]

ros2 pkg executables turtlesim

上述命令应返回turtlesim的可执行文件列表: [待校准@8439]

turtlesim draw_square
turtlesim mimic
turtlesim turtle_teleop_key
turtlesim turtlesim_node

2启动turtlesim [待校准@8440]

要启动turtlesim，请在终端中输入以下命令: [待校准@8441]

ros2 run turtlesim turtlesim_node

模拟器窗口应该出现，中间有一只随机的乌龟。 [待校准@8442]

在命令下的终端中，您将看到来自节点的消息: [待校准@8443]

[INFO] [turtlesim]: Starting turtlesim with node name /turtlesim

[INFO] [turtlesim]: Spawning turtle [turtle1] at x=[5.544445], y=[5.544445], theta=[0.000000]

在这里你可以看到你的默认海龟的名字是 turtle1 ，以及它产生的默认坐标。 [待校准@8444]

3使用turtlesim [待校准@8445]

再次打开新终端和源文件ROS 2。 [待校准@8446]

现在，您将运行一个新节点来控制第一个节点中的海龟: [待校准@8447]

ros2 run turtlesim turtle_teleop_key

此时，你应该打开三个窗口: 一个运行 turtlesim_node 的终端，一个运行 turtle_teleop_key 的终端和turtlesim窗口。安排这些窗口，这样你就可以看到海龟的窗口，也可以激活运行 turtle_teleop_key 的终端，这样你就可以控制海龟。 [待校准@8448]

使用键盘上的箭头键来控制海龟。它将在屏幕上移动，使用其附带的 "pen" 绘制到目前为止的路径。 [待校准@8449]

备注

按箭头键只会使乌龟移动一小段距离然后停止。这是因为，realisti调用y，例如，如果操作员失去了与机器人的连接，您不希望机器人继续执行指令。 [待校准@8450]

您可以使用 list 命令查看节点及其关联的服务、话题和动作: [待校准@8451]

ros2 node list
ros2 topic list
ros2 service list
ros2 action list

您将在接下来的教程中了解更多关于这些概念的信息。由于本教程的目标只是获得turtlesim的一般概述，我们将使用rqt (ROS 2的图形用户界面) 来仔细研究服务。 [待校准@8452]

4安装rqt [待校准@8453]

打开一个新的终端来安装 rqt 及其插件: [待校准@8454]

sudo apt update

sudo apt install ~nros-foxy-rqt*

sudo apt update

sudo apt install ros-foxy-rqt*

标准archive安装ROS 2 macOS包含 rqt 及其插件，所以你应该已经 rqt 安装。 [待校准@8457]

标准archive安装ROS 2 Windows包含 rqt 及其插件，所以你应该已经 rqt 安装。 [待校准@8458]

运行rqt: [待校准@8459]

rqt

5使用rqt [待校准@8460]

第一次运行rqt后，窗口将为空白。不用担心; 只需从顶部的菜单栏中选择插件> 服务呼叫者。 [待校准@8461]

备注

rqt可能需要一些时间来找到所有插件本身。如果你点击插件，但没有看到服务或任何其他选项，你应该关闭rqt，在你的终端输入命令 rqt --force-discover 。 [待校准@8462]

使用服务下拉列表左侧的刷新按钮，确保turtlesim节点的所有服务都可用。 [待校准@8463]

单击服务下拉列表以查看turtlesim的服务，然后选择 /spawn 服务。 [待校准@8464]

5.1尝试生成服务 [待校准@8465]

让我们使用rqt调用 /spawn 服务。你可以从它的名字中猜到 /spawn 会在turtlesim窗口中创造出另一只乌龟。 [待校准@8466]

给新乌龟唯一名称，像 turtle2 的double击之间空单引号表达立柱。您可以看到此表达式对应于名称值，并且类型为string。 [待校准@8467]

输入海龟产卵的新坐标，如 x = 1.0 和 y = 1.0 。 [待校准@8468]

备注

如果您尝试生成与现有海龟同名的新海龟，如您的默认 turtle1 ，您将在运行 turtlesim_node 的终端中收到一条错误消息: [待校准@8469]

[ERROR] [turtlesim]: A turtle named [turtle1] already exists

要生成turtle2，您必须通过单击rqt窗口右上角的呼叫按钮来调用服务。 [待校准@8470]

你会看到一只新的乌龟 (同样是随机设计的) 在你为x和y输入的坐标处产卵。 [待校准@8471]

如果您在rqt中刷新服务列表，您还将看到除了 “/turtle1/…” 之外，现在还有与新turtle “/turtle2/…” 相关的服务。 [待校准@8472]

5.2尝试set_pen服务 [待校准@8473]

现在，让我们使用 /set_pen 服务为turtle1提供独特的笔: [待校准@8474]

r、g和b的值在0到255之间，将设置turtle1绘制的笔的颜色，宽度设置线的厚度。 [待校准@8475]

要让turtle1绘制一条不同的红线，请将r的值更改为255，将宽度的值更改为5。更新值后，不要忘记调用服务。 [待校准@8476]

如果你回到 turtle_teleop_node 运行的终端，按下箭头键，你会看到turtle1的笔变了。 [待校准@8477]

你've probably noticed that there'没有办法移动turtle2。你可以通过将turtle1的 cmd_vel 话题重新映射到turtle2上来实现这一点。 [待校准@8478]

6重新映射 [待校准@7277]

在新终端中，源文件ROS 2，然后运行: [待校准@8479]

ros2 run turtlesim turtle_teleop_key --ros-args --remap turtle1/cmd_vel:=turtle2/cmd_vel

现在，您可以在该终端处于活动状态时移动turtle2，在运行 turtle_teleop_key 的另一个终端处于活动状态时移动turtle1。 [待校准@8480]

7关闭turtlesim [待校准@8481]

要停止仿真，您可以在 turtlesim_node 终端输入 “ctrl + c”，在远程终端输入 q 。 [待校准@8482]

总结 [待校准@6307]

使用turtlesim和rqt是学习ROS 2核心概念的好方法。 [待校准@8483]
