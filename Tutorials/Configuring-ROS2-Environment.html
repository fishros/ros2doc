配置ROS 2环境 [待校准@7086]

Goal目标: This本教程将向您展示如何准备ROS 2环境。 [待校准@7087]

Tutorial教程级别: Beginner初学者 [待校准@7088]

Time时间: 5 5分钟 [待校准@6714]

背景 [待校准@6716]

ROS 2依赖于使用shell环境组合工作区的概念。 "Workspace" 是一个ROS术语，表示您在系统中使用ROS 2进行开发的位置。核心ROS 2工作区被调用为底层。后续本地工作区将调用ed覆盖。使用ROS 2进行开发时，您将典型地调用y同时激活多个工作区。 [待校准@7089]

结合工作空间使针对不同版本的ROS 2或针对不同包的开发变得更加容易。它还允许在同一台计算机上安装多个ROS 2发行版 (或 “发行版”，例如Dashing和Eloquent) 并在它们之间切换。 [待校准@7090]

这是通过每次打开新shell时采购安装程序文件或将源文件命令添加到shell启动脚本一次来完成的。如果不采购安装文件，您将无法访问ROS 2命令，也无法查找或使用ROS 2包。换句话说，您将无法使用ROS 2。 [待校准@7091]

先决条件 [待校准@6718]

在开始这些教程之前，请按照ROS 2 安装 页面上的说明安装ROS 2。 [待校准@7092]

命令用于本教程假设你跟着二进制包安装指南操作系统 (Debian包Linux)。你还是可以如果你由源文件，但路径设置文件可能会不同。您也无法使用 sudo apt install ros-<distro>-<package> 命令 (常用初学者级别教程) 如果你安装源文件。 [待校准@7093]

如果您使用的是Linux或macOS，但还不熟悉外壳， this tutorial 将会有所帮助。 [待校准@7094]

任务 [待校准@6725]

1获取安装文件 [待校准@7095]

您需要在打开的每个新shell上运行此命令才能访问ROS 2命令，如下所示: [待校准@7096]

source /opt/ros/foxy/setup.bash

. ~/ros2_install/ros2-osx/setup.bash

call C:\dev\ros2\local_setup.bat

注解

确切的命令取决于您安装ROS 2的位置。如果遇到问题，请确保文件路径导致安装。 [待校准@7097]

2将采购添加到您的shell启动脚本 [待校准@7098]

如果您不想每次打开新shell时都必须源文件 (跳过任务1)，则可以将命令添加到shell启动脚本中: [待校准@7099]

echo "source /opt/ros/foxy/setup.bash" >> ~/.bashrc

撤消，找到系统的shell启动脚本并删除附加源文件命令。 [待校准@7100]

echo "source ~/ros2_install/ros2-osx/setup.bash" >> ~/.bash_profile

仅适用于PowerShell用户，在'My Documents'调用ed'WindowsPowerShell'中创建一个文件夹。在'WindowsPowerShell'内，创建文件 “microsoft.PowerShell_profile.ps1”。在文件内部，粘贴: [待校准@7101]

C:\dev\ros2_foxy\local_setup.ps1

每次打开新shell时，PowerShell都会请求运行此脚本的权限。为避免该问题，您可以运行: [待校准@7102]

Unblock-File C:\dev\ros2_foxy\local_setup.ps1

要在Linux和macOS中撤消此操作 (更改为另一个发行版)，请找到系统的shell启动脚本并删除附加的源文件命令。 [待校准@7103]

3检查环境变量 [待校准@7104]

采购ROS 2安装文件将设置几个操作ROS 2所需的环境变量。如果您在查找或使用ROS 2包时遇到问题，请确保使用以下命令正确设置了环境: [待校准@7105]

printenv | grep -i ROS

printenv | grep -i ROS

set | findstr -i ROS

检查是否设置了像 ROS_DISTRO 和 ROS_VERSION 这样的变量。例如，如果你正在使用Foxy，你会看到: [待校准@7106]

ROS_VERSION=2
ROS_PYTHON_VERSION=3
ROS_DISTRO=foxy

如果环境变量设置不正确，请返回您遵循的安装指南的ROS 2包安装部分。如果你需要更具体的帮助 (因为环境设置文件可以来自不同的地方)，你可以从社区 get answers 。 [待校准@7107]

3.1 ROS_DOMAIN_ID 变量 [待校准@7108]

有关ROS域id的详细信息，请参阅 domain ID 文章。 [待校准@7109]

确定ROS 2代理组的唯一整数后，可以使用以下命令设置环境变量: [待校准@7110]

export ROS_DOMAIN_ID=<your_domain_id>

要在shell会话之间维护此设置，您可以将命令添加到shell启动脚本中: [待校准@7111]

echo "export ROS_DOMAIN_ID=<your_domain_id>" >> ~/.bashrc

export ROS_DOMAIN_ID=<your_domain_id>

要在shell会话之间维护此设置，您可以将命令添加到shell启动脚本中: [待校准@7111]

echo "export ROS_DOMAIN_ID=<your_domain_id>" >> ~/.bash_profile

set ROS_DOMAIN_ID=<your_domain_id>

如果要在shell会话之间使此永久保留，请同时运行: [待校准@7112]

setx ROS_DOMAIN_ID <your_domain_id>

总结 [待校准@6747]

ROS 2开发版本环境需要正确配置使用前。有两种方式: 采购安装文件在每个新外壳打开，或添加源文件命令启动脚本。 [待校准@7113]

如果您在使用ROS 2定位或使用包时遇到任何问题，您应该做的第一件事是检查您的环境变量，并确保它们设置为您想要的版本和发行版。 [待校准@7114]
