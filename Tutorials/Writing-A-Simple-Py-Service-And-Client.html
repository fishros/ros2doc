背景 [待校准@6276]

当 nodes 使用 services 进行通信时，发送数据请求的节点被调用到客户端节点，响应请求的节点是服务节点。请求和响应的结构由 .srv 文件决定。 [待校准@8955]

这里使用的示例是一个简单的整数加法系统; 一个节点请求两个整数的总和，另一个节点响应结果。 [待校准@8956]

先决条件 [待校准@6278]

在之前的教程中，你学习了如何 create a workspace and create a package 。 [待校准@8909]

1创建包 [待校准@6697]

打开一个新的终端和 source your ROS 2 installation ，这样 ros2 命令就可以工作了。 [待校准@8774]

导航到 dev_ws 目录创建 previous tutorial 。 [待校准@8775]

重新调用应该在 src 目录中创建的包，而不是工作区的根目录。导航到 dev_ws/src 并创建一个新的包: [待校准@8776]

ros2 pkg create --build-type ament_python py_srvcli --dependencies rclpy example_interfaces

您的终端将返回一条消息，验证您的包 py_srvcli 及其所有必要的文件和文件夹的创建。 [待校准@9030]

[需手动修复的语法] --dependencies 参数将自动调用y将必要的依赖关系添加到 package.xml 。 example_interfaces 是包含 the .srv file 的套餐，您需要构建您的请求和回复: [待校准@9031]

int64 a
int64 b
---
int64 sum

前两行是请求的参数，破折号下面是响应。 [待校准@8959]

2写入服务节点 [待校准@8960]

在 dev_ws/src/py_srvcli/py_srvcli 目录内，创建一个调用 service_member_function.py 的新文件，并将以下代码粘贴到: [待校准@9035]

from example_interfaces.srv import AddTwoInts

import rclpy
from rclpy.node import Node


class MinimalService(Node):

    def __init__(self):
        super().__init__('minimal_service')
        self.srv = self.create_service(AddTwoInts, 'add_two_ints', self.add_two_ints_callback)

    def add_two_ints_callback(self, request, response):
        response.sum = request.a + request.b
        self.get_logger().info('Incoming request\na: %d b: %d' % (request.a, request.b))

        return response


def main(args=None):
    rclpy.init(args=args)

    minimal_service = MinimalService()

    rclpy.spin(minimal_service)

    rclpy.shutdown()


if __name__ == '__main__':
    main()

第一个 import 语句从 example_interfaces 包中导入 AddTwoInts 服务类型。以下 import 语句导入ROS 2 Python客户端库，并指定调用 Node 类。 [待校准@9036]

from example_interfaces.srv import AddTwoInts

import rclpy

[需手动修复的语法] MinimalService 类构造函数初始化名称为 minimal_service 的节点。然后，它创建一个服务并定义类型、名称和调用。 [待校准@9037]

def __init__(self):

服务调用back的定义接收请求数据，对其求和，并将总和作为响应返回。 [待校准@9038]

3写入客户端节点 [待校准@8973]

在 dev_ws/src/py_srvcli/py_srvcli 目录内，创建一个调用 client_member_function.py 的新文件，并将以下代码粘贴到: [待校准@9042]

import sys

from example_interfaces.srv import AddTwoInts
import rclpy
from rclpy.node import Node


class MinimalClientAsync(Node):

    def __init__(self):
        super().__init__('minimal_client_async')
        self.cli = self.create_client(AddTwoInts, 'add_two_ints')
        while not self.cli.wait_for_service(timeout_sec=1.0):
            self.get_logger().info('service not available, waiting again...')
        self.req = AddTwoInts.Request()

    def send_request(self):
        self.req.a = int(sys.argv[1])
        self.req.b = int(sys.argv[2])
        self.future = self.cli.call_async(self.req)


def main(args=None):
    rclpy.init(args=args)

    minimal_client = MinimalClientAsync()
    minimal_client.send_request()

    while rclpy.ok():
        rclpy.spin_once(minimal_client)
        if minimal_client.future.done():
            try:
                response = minimal_client.future.result()
            except Exception as e:
                minimal_client.get_logger().info(
                    'Service call failed %r' % (e,))
            else:
                minimal_client.get_logger().info(
                    'Result of add_two_ints: for %d + %d = %d' %
                    (minimal_client.req.a, minimal_client.req.b, response.sum))
            break

    minimal_client.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()

3.1检查代码 [待校准@8935]

对客户来说，唯一不同的 import 声明是 import sys 。客户端节点代码使用 sys.argv 来访问请求的命令行输入参数。 [待校准@9043]

构造函数定义创建与服务节点具有相同类型和名称的客户端。类型和名称必须匹配，客户端和服务才能进行通信。 [待校准@9044]

构造函数中的 while 循环检查与客户端类型和名称匹配的服务是否每秒可用一次。 [待校准@9045]

构造函数下方是请求定义，后跟 main 。 [待校准@9046]

客户的 main 唯一显著的区别是 while 环。只要系统正在运行，循环就会检查 future ，看看是否有来自服务的响应。如果服务已发送响应，则结果将写入日志消息中。 [待校准@9047]

3.2添加入口点 [待校准@9023]

与服务节点一样，您还必须添加一个入口点才能运行客户端节点。 [待校准@9048]

[需手动修复的语法] setup.py 文件的 entry_points 字段应该如下所示: [待校准@9049]

entry_points={
    'console_scripts': [
        'service = py_srvcli.service_member_function:main',
        'client = py_srvcli.client_member_function:main',
    ],
},
