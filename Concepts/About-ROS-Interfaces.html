1.背景 [待校准@5527]

ROS应用程序类型调用y通过三种类型之一的接口进行通信: 消息、服务和动作。ROS 2使用简化的描述语言，即接口定义语言 (IDL) 来描述这些接口。此描述使ROS工具可以轻松地以多种目标语言自动调用y生成接口类型的源文件。 [待校准@5528]

在本文档中，我们将描述支持的类型。 [待校准@5529]

  • msg: .msg 文件是描述ROS消息字段的简单文本文件。它们用于为不同语言的消息生成源文件代码。 [待校准@5530]

  • srv: .srv 文件描述了一项服务。它们由两部分组成: 请求和响应。请求和响应是消息声明。 [待校准@5531]

  • 动作: .action 文件描述动作。它们由三部分组成: 目标、结果和反馈。每个部分本身都是一个消息声明。 [待校准@5532]

2.消息描述规范 [待校准@5533]

消息描述和定义在 .msg 文件 msg/ 目录ROS包。 .msg 文件由两部分组成: 字段和常量。 [待校准@5534]

2.1字段 [待校准@5535]

每个字段由一个类型和一个名称组成，用一个空格分隔，即: [待校准@5536]

fieldtype1 fieldname1
fieldtype2 fieldname2
fieldtype3 fieldname3

例如: [待校准@5537]

int32 my_int
string my_string

  • 内置类型 [待校准@5540]

2.1.2字段名称 [待校准@5603]

字段名必须是小写字母数字字符，并带有下划线，用于分隔单词。它们必须以字母字符开头，不能以下划线结尾，也不能有两个连续的下划线。 [待校准@5604]

2.1.3字段默认值 [待校准@5605]

默认值可以设置为消息类型中的任何字段。目前，string阵列和复杂类型不支持默认值 (即上述内置类型表中不存在的类型，适用于所有嵌套消息) [待校准@5606]

定义默认值是通过向字段定义行添加第三个元素来完成的，即: [待校准@5607]

fieldtype fieldname fielddefaultvalue

例如: [待校准@5537]

uint8 x 42
int16 y -2000
string full_name "John Doe"
int32[] samples [-200, -100, 0, 100, 200]

注意: [待校准@5608]

  • [需手动修复的语法]string值必须在单个 ' `` or double quotes `` "' 中定义 [待校准@5609]

  • 目前string值未转义 [待校准@5610]

2.2常数 [待校准@5611]

每个常量定义都类似于具有默认值的字段描述，不同之处在于该值永远无法更改programati调用y。该值分配通过使用等号 “=” 来指示，例如 [待校准@5612]

constanttype CONSTANTNAME=constantvalue

例如: [待校准@5537]

int32 X=123
int32 Y=-123
string FOO="foo"
string EXAMPLE='bar'

备注

常量名称必须为大写 [待校准@5613]

3.服务描述规范 [待校准@5614]

服务描述和定义在 .srv 文件 srv/ 目录ROS包。 [待校准@5615]

服务描述文件由请求和响应类型 msg 组成，以 “---” 分隔。任何两个与 “---” 连接的 .msg 文件都是法律服务描述。 [待校准@5616]

这是一个非常简单的服务示例，该服务接受string并返回string: [待校准@5617]

string str
---
string str

我们当然会变得更加复杂 (如果您想引用来自同一包的消息，您不能提及包名称): [待校准@5618]

#request constants
int8 FOO=1
int8 BAR=2
#request fields
int8 foobar
another_pkg/AnotherMessage msg
---
#response constants
uint32 SECRET=123456
#response fields
another_pkg/YetAnotherMessage val
CustomMessageDefinedInThisPackage value
uint32 an_integer

您不能在服务中嵌入其他服务。 [待校准@5619]

4.ROS 2接口的新功能 [待校准@5620]

ROS 2 IDL与 ROS 1 IDL 密切相关。大多数现有的ROS 1 .msg 和 .srv 文件应该与ROS 2一样可用。在ROS 1现有的特征集上，ROS 2 IDL引入了一些新的特征，即: [待校准@5621]

  • 有界数组: 而ROS 1IDL允许无界数组 (例如g.， int32[] foo ) 和固定尺寸阵列 (例如g.， int32[5] bar )，ROS 2 IDL进一步允许有界阵列 (例如g.， int32[<=5] bat )。在某些用例中，重要的是能够在数组大小上设置上限，而不必总是使用那么多空间 (例如g.，在实时系统中，您需要预分配将在执行期间使用的所有内存)。 [待校准@5622]

  • 有界strings: 而ROS 1IDL允许无界strings (例如， string foo )，而ROS 2 IDL进一步允许有界strings (例如， string<=5 bar )。 [待校准@5623]

  • 默认值: 而ROS 1IDL允许恒定字段 (例如， int32 X=123 )，而ROS 2 IDL进一步允许指定默认值 (例如， int32 X 123 )。在构造消息/服务对象时使用默认值，随后可以通过分配给字段来覆盖默认值。您还可以为动作部分指定默认值。 [待校准@5624]
