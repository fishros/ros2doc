概述 [待校准@5142]

客户端库是允许用户实现其ROS代码的api。使用客户端库，用户可以访问ROS概念，如节点、话题、服务等。客户端库具有多种编程语言，因此用户可以使用最适合其应用程序的语言编写ROS代码。例如，您可能更喜欢用Python编写可视化工具，因为它可以使原型迭代更快，而对于系统中与效率有关的部分，节点可以更好地在C++ 中实现。 [待校准@5460]

使用不同客户端库编写的节点能够相互共享消息，因为所有客户端库都实现了代码生成器，使用户能够以各自的语言与ROS接口文件进行交互。 [待校准@5461]

除了特定语言的通讯工具外，客户端库还向用户公开使ROS成为 “ROS” 的核心功能。例如，以下是可通过客户端库典型调用y的功能列表: [待校准@5462]

  • 名称和命名空间 [待校准@5463]

  • 时间 (真实的或模拟的) [待校准@5464]

  • 参数 [待校准@5409]

  • 控制台日志 [待校准@5465]

  • 螺纹模型 [待校准@5466]

  • 进程内通讯 [待校准@5467]

支持的客户端库 [待校准@5468]

Cclient客户端库 ( rclcpp ) 和Python客户端库 ( rclpy ) 都是利用RCL中通用功能的客户端库。 [待校准@5469]

虽然C++ 和Python客户端库由核心ROS 2团队维护，但ROS 2社区的成员创建了其他客户端库: [待校准@5470]

  • JVM and Android

  • Objective C and iOS

  • C#

  • Swift

  • Node.js

  • Ada

  • _.NET Core, UWP and C#

  • Rust

常见功能: RCL [待校准@5479]

客户端库中的大多数功能并不特定于客户端库的编程语言。例如，理想情况下，参数的行为和命名空间的逻辑在所有编程语言中应该是相同的。因此，客户端库不是从头开始实现通用功能，而是使用通用核心ROS客户端库 (RCL) 实现非特定语言的ROS概念的逻辑和行为的接口。因此，客户端库只需要用外部函数接口包装RCL中的通用功能。这使得客户端库更薄，更易于开发。因此，常见的RCL功能通过C接口公开，因为C语言通常是调用客户端库最简单的语言。 [待校准@5480]

除了使客户端库轻量级之外，具有共同核心的一个优点是语言之间的行为更加一致。如果对核心RCL (例如命名空间) 中的功能的逻辑/行为进行了任何更改，则所有使用RCL的客户端库都将反映这些更改。此外，拥有共同的核心意味着在错误修复方面，维护多个客户端库变得不那么容易。 [待校准@5481]

The API documentation for the RCL can be found here.

特定语言功能 [待校准@5483]

需要特定于语言的功能/属性的客户端库概念不在RCL中实现，而是在每个客户端库中实现。例如，“spin” 函数使用的线程模型将具有特定于客户端库语言的实现。 [待校准@5484]

与ROS 1的比较 [待校准@5396]

在ROS 1，所有的客户库都是由 "from the ground up" 开发的。这允许ROS 1Python客户端库执行纯粹Python例如，带来的好处不需要编译代码。但是，客户端库之间的命名约定和行为并不总是一致的，必须在多个地方进行错误修复，并且有许多仅在一个客户端库中实现的功能 (例如g. UDPROS)。 [待校准@5487]

总结 [待校准@5488]

通过使用通用核心ROS客户端库，用多种编程语言编写的客户端库更易于编写，行为更一致。 [待校准@5489]
