概述 [待校准@5142]

ROS中的参数与单个节点相关联。参数用于在启动时 (以及运行时) 配置节点，而无需更改代码。参数的生存期与节点的生存期相关联 (尽管节点可以实现某种持久性以在重新启动后重新加载值)。 [待校准@5491]

参数按节点名称、节点命名空间、参数名称和参数命名空间寻址。提供参数命名空间是可选的。 [待校准@5492]

每个参数由一个键、一个值和一个描述符组成。键是string，值是以下类型之一: bool、int64、float64、string、字节 [] 、bool[] 、int64[] 、float64[] 或string[]。默认情况下，所有描述符均为空，但可以包含参数描述、值范围、类型信息和其他约束。 [待校准@5493]

有关ROS参数的动手教程，请参阅: doc:'//教程/参数/Understanding-ROS2-Parameters'。 [待校准@5494]

参数背景 [待校准@5495]

声明参数 [待校准@5496]

默认情况下，节点需要 * 声明 * 在其生命周期内将接受的所有参数。这样可以在节点启动时很好地定义参数的类型和名称，从而减少以后配置错误的机会。请参阅: 文档: 'Tutorials/教程/在类中使用参数-cpp' 或: 文档: 'Tutorials/教程/在类中使用参数-python' 用于声明和使用节点参数的教程。 [待校准@5497]

对于某些类型的节点，并非所有参数都会提前知道。在这些情况下，可以使用将 allow_undeclared_parameters 设置为 true 来实例化节点，这将允许在节点上获取和设置参数，即使它们尚未声明。 [待校准@5498]

参数类型 [待校准@5499]

ROS 2节点上的每个参数都具有概述中提到的预定义参数类型之一。默认情况下，在运行时尝试更改声明参数的类型将失败。这可以防止常见错误，例如将boolean值放入整数参数中。 [待校准@5500]

如果一个参数需要是多个不同的类型，并且使用该参数的代码可以处理它，则可以更改此默认行为。声明该参数时，应使用 dynamic_typing 成员变量设置为 true 的 ParameterDescriptor 进行声明。 [待校准@5501]

参数调用返回 [待校准@5502]

ROS 2节点可以注册两种不同类型的调用，以便在参数发生更改时通知。有两种类型的调用返回的原因是为了有机会在参数更改发生之前进行干预，以及有机会在参数更改发生之后做出反应。节点可以同时注册调用的back类型之一，也可以不注册。这两种类型如下所述。 [待校准@5503]

第一种类型被称为 "set parameter" 调用，可以通过调用 add_on_set_parameters_callback 来安装。调用back应该接受一个 Parameter 对象列表，并返回一个 rcl_interfaces/msg/SetParametersResult 。在节点上声明或更改参数之前，将调用此调用。调用back的主要目的是使用户能够检查即将对参数进行的更改，并明确拒绝该更改。 [待校准@5504]

备注

重要的是 "set parameter" 调用没有副作用。由于可以链接多个 "set parameter" 调用，因此单个调用无法知道以后的调用是否会拒绝更新。例如，如果单个调用要对其所在的类进行更改，它可能会与实参不同步。要在成功更改参数后 * 返回调用，请参阅下面的下一种调用类型。 [待校准@5505]

第二种类型的调用被称为 "on parameter event" 调用，可以通过调用 on_parameter_event 来安装。调用back应该接受一个 rcl_interfaces/msg/ParameterEvent 对象，并且不返回任何内容。在声明、更改或删除所有参数后，将调用此调用。调用back的主要目的是使用户能够对已成功接受的参数的更改做出反应。 [待校准@5506]

与参数交互 [待校准@5507]

ROS 2节点可以通过节点api执行参数操作，如: doc:'//教程/使用-Class-cpp中的参数' 或: doc:'Tutorials/教程/在类中使用参数-python'。外部进程可以通过在实例化节点时默认创建的参数服务来执行参数操作。默认情况下创建的服务为: [待校准@5508]

  • /node_name/describe_parameters: 使用 rcl_interfaces/srv/DescribeParameters 的服务类型。给定参数名称列表，返回与参数关联的描述符列表。 [待校准@5509]

  • /node_name/get_parameter_types: 使用 rcl_interfaces/srv/GetParameterTypes 的服务类型。给定参数名称列表，返回与参数关联的参数类型列表。 [待校准@5510]

  • /node_name/get_parameters: 使用 rcl_interfaces/srv/GetParameters 的服务类型。给定参数名称列表，返回与参数关联的参数值列表。 [待校准@5511]

  • /node_name/list_parameters: 使用 rcl_interfaces/srv/ListParameters 的服务类型。给定参数前缀的可选列表，返回具有该前缀的可用参数的列表。如果前缀为空，则返回所有参数。 [待校准@5512]

  • /node_name/set_parameters: 使用 rcl_interfaces/srv/SetParameters 的服务类型。给定参数名称和值列表，尝试在节点上设置参数。返回尝试设置每个参数的结果列表; 其中一些可能成功，一些可能失败。 [待校准@5513]

  • /node_name/set_parameters _ atomi调用y: 使用 rcl_interfaces/srv/SetParametersAtomically 的服务类型。给定参数名称和值列表，尝试在节点上设置参数。尝试设置所有参数返回单个结果，因此如果一个参数失败，则所有参数都失败。 [待校准@5514]

运行节点时设置初始参数值 [待校准@5515]

在运行节点时，可以通过单个命令行参数或通过YAML文件设置初始参数值。有关如何设置初始参数值的示例，请参见 直接从命令行设置参数 [待校准@762] 。 [待校准@5516]

ros2 run demo_nodes_cpp parameter_blackboard
