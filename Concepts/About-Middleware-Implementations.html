关于ROS 2中间件实现 [待校准@5347]

ROS中间件实现是一组 | 包 |，用于实现一些内部ROS接口，例如 rmw 、 rcl 和 rosidl | Api |。 [待校准@5348]

DDS中间件包通用包 [待校准@5349]

当前所有ROS中间件实现均基于全部或部分DDS实现。例如，有一个使用RTI's Connext DDS and an implementation which uses eProsima'Fast DDS的中间件实现。因此，在大多数基于DDS的中间件实现中，有一些共享的 | 包 |。 [待校准@5350]

在 | GitHub | _ 上的 ros2/rosidl_dds 仓库中，有以下 | package |: [待校准@5351]

  • rosidl_generator_dds_idl: provides tools to generate DDS .idl files from rosidl files, e.g. .msg files, .srv files等 [待校准@5352]

[需手动修复的语法] rosidl_generator_dds_idl | 包 | 为每个 rosidl 文件生成DDS .idl 文件，例如 .msg 文件，由 | 包 | 定义，包含消息。目前，基于DDS的ROS中间件实现利用该生成器的输出 .idl 文件来生成针对供应商的预编译类型支持。 [待校准@5353]

ROS中间件实现的结构 [待校准@5354]

ROS中间件实现的典型调用y由单个仓库中的几个 | 包 | 组成: [待校准@5355]

  • [需手动修复的语法]``<implementation_name>_cmake_module``:包含CMake模块，用于发现和公开所需的依赖关系 [待校准@5356]

  • [需手动修复的语法]``rmw_<implementation_name>_<language>``: contains the implementation of the rmw | API | 在特定语言中，类型调用y c ++ [待校准@5357]

  • [需手动修复的语法]``rosidl_typesupport_<implementation_name>_<language>``: contains tools to generate static type support code for rosidl files，根据特定语言的实现量身定制，典型地调用y C或C ++ [待校准@5358]

[需手动修复的语法] <implementation_name>_cmake_module | package | 包含查找中间件实现的支持依赖项所需的任何CMake模块和函数。例如， rti_connext_dds_cmake_module 围绕RTI Connext DDS附带的CMake模块提供包装逻辑，以确保所有依赖于它的包将选择相同的RTI Connext DDS安装。类似地， fastrtps_cmake_module 包含一个CMake模块来查找eProsima的Fast DDS， gurumdds_cmake_module 包含一个CMake模块来查找GurumNetworks GurumDDS。并非所有实现都将具有这样的包: 例如，Eclipe的CycloneDDS已经提供了一个CMake模块，该模块直接由其RMW实现使用，而无需额外的包装器。 [待校准@5359]

[需手动修复的语法] rmw_<implementation_name>_<language> | 包 | 在特定语言中实现 rmw C | API |。实现本身可以是C++，它只需要将标题的符号公开为 “外部 "C" ”，这样C应用程序就可以链接到它。 [待校准@5360]

[需手动修复的语法] rosidl_typesupport_<implementation_name>_<language> | 包 | 提供一个生成器，该生成器以特定语言生成DDS代码。这是使用由 rosidl_generator_dds_idl | package | 生成的 .idl 文件和DDS供应商提供的DDS IDL代码生成器来完成的。它还生成用于在DDS消息结构之间转换ROS消息结构的代码。该生成器还负责为其正在使用的消息包创建共享库，该共享库特定于消息包中的消息和正在使用的DDS供应商。 [待校准@5361]

如上所述，如果rmw实现支持消息的运行时解释，则可以使用 rosidl_typesupport_introspection_<language> 代替供应商特定类型的支持包。通过支持 DDS X-Types Dynamic Data standard 实现了通过话题编程调用y发送和接收类型而无需事先生成代码的能力。因此，rmw实现可以为X-type标准提供支持，和/或为在编译时生成的特定于其DDS实现的类型支持提供包。 [待校准@5362]

作为rmw实现仓库的一个示例， Eclipse Cyclone DDS ROS中间件实现位于 | GitHub | _ at ros2/rmw_cyclonedds 乙。 [待校准@5363]

[需手动修复的语法] Fast DDS 的rmw实现在 | GitHub | _ at ros2/rmw_fastrtps_cpp 。 [待校准@5364]

[需手动修复的语法] Connext DDS 的rmw实现在 | GitHub | _ at ros2/rmw_connextdds 。 [待校准@5365]

[需手动修复的语法] GurumDDS 的rmw实现在 | GitHub | _ at ros/rmw_gurumdds 。 [待校准@5366]
