概述 [待校准@5142]

当前支持的日志功能是: [待校准@5326]

  • 客户端库 ( rclcpp 和 rclpy ) 使用通用日志库提供: [待校准@5327]

    • 带有各种过滤器的日志调用。 [待校准@5328]

    • 日志记录器的层次结构 [待校准@5329]

    • 与自动调用的节点相关联的日志记录器使用节点的名称和命名空间。 [待校准@5330]

  • 控制台输出。 [待校准@5331]

    • 文件输出和类似于 rosout 的功能即将到来，用于远程消息消费。 [待校准@5332]

  • 日志记录器级别的编程配置。 [待校准@5333]

    • 支持默认日志记录器级别的Launch时配置; 即将发布配置文件和运行时的外部配置。 [待校准@5334]

日志记录器概念 [待校准@5335]

日志消息具有与之相关的严重性级别: DEBUG 、 INFO 、 WARN 、 ERROR 或 FATAL ，按升序排列。 [待校准@5336]

日志记录器将仅处理严重性等于或高于为日志记录器选择的指定级别的日志消息。 [待校准@5337]

每个节点 (在 rclcpp 和 rclpy 中) 都有一个与之关联的日志记录器，自动调用y包括节点的名称和命名空间。如果节点的名称在外部重新映射到源文件中定义的名称以外的其他名称，则它将反映在日志记录器名称中。还可以创建使用特定名称的非节点日志记录器。 [待校准@5338]

日志记录器名称表示层次结构。如果名为 “abc.de f” 的日志记录器的级别未设置，它将服从其父级名为 "abc" 的级别，如果该级别也未设置，将使用默认的日志记录器级别。当日志记录器 "abc" 的水平改变时，它的所有后代 (例如g.“abc.de f”，“abc。ghi。jkl”) 将影响其级别，除非已明确设置其级别。 [待校准@5339]

日志使用情况 [待校准@5340]

  • 例如用法，请参见 logging demo 。 [待校准@5342]

  • 有关功能的广泛列表，请参阅 rclcpp documentation 。 [待校准@5343]

  • 有关节点日志记录器的使用情况，请参见 rclpy examples 。 [待校准@5345]

  • 例如，关键词参数的使用 (例如， skip_first ， once ) 见 rclpy tests 。 [待校准@5346]
